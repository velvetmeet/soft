## Деплой на Cloudflare Pages

### Способ 1 — Drag & Drop (быстрый старт)

1. Зайди на https://dash.cloudflare.com (бесплатный аккаунт)
2. Workers & Pages → Create → Pages → Upload assets
3. Project name: `velvet-soft` (или любое другое)
4. Перетащи всю папку (со всеми файлами и подпапками) в зону загрузки
5. Deploy site
6. Через 30-60 секунд получишь URL: `https://velvet-soft.pages.dev`

### Способ 2 — Через GitHub (для регулярных правок)

1. Создай приватный репозиторий на GitHub
2. Залей туда все файлы (`index.html`, `_redirects`, `functions/`, `images/`)
3. Cloudflare Pages → Create project → Connect to Git → выбери репозиторий
4. Production branch: `main`
5. Build settings: оставь пустыми (это не SPA, не нужна сборка)
6. Save and Deploy

После любого `git push` сайт автоматически обновляется за 30-60 секунд.

### Проверка после деплоя

Открой в браузере:
- `https://velvet-soft.pages.dev` → должен открыться gate на английском
- `https://velvet-soft.pages.dev/?lang=de` → на немецком
- `https://velvet-soft.pages.dev/?lang=pl` → на польском
- `https://velvet-soft.pages.dev/?s=test-source` → откроется gate, после клика — редирект на партнёрку с меткой `aff_sub5=test-source`

## Настройка партнёрской ссылки

Партнёрская ссылка хранится в `functions/go.js` в константе `AFFILIATE_BASE`:

```javascript
const AFFILIATE_BASE = "https://go.cm-trk6.com/aff_f?h=rA47F6";
```

Если сменилась кампания, оффер или хэш — меняешь только эту строку. Не трогай `&aff_sub5=` — он добавляется автоматически из `?s=` параметра.

После изменения — push в репозиторий или re-deploy.

## Tracking трафика по источникам

### Как работает

В URL добавляешь параметр `?s=метка`. Например:
- `https://velvet-soft.pages.dev/?s=ig-anna_de`
- `https://velvet-soft.pages.dev/?s=fb-page_pl`
- `https://velvet-soft.pages.dev/?s=yt-rain_de`

При клике на кнопку юзер идёт на `/go?s=ig-anna_de` → серверный 302 на `go.cm-trk6.com/aff_f?h=...&aff_sub5=ig-anna_de`.

В партнёрке `cm-trk6.com` в колонке `aff_sub5` ты увидишь свою метку. Можешь фильтровать статистику по любой метке.

### Правила именования меток

Формат: `платформа-аккаунт_язык`

Примеры:
- `ig-anna_de` — Instagram, аккаунт Anna, немецкий язык
- `fb-mindful_pl` — Facebook страница «Mindful», польский
- `yt-rain_de` — YouTube канал «Rain», немецкий
- `tt-acc1_en` — TikTok аккаунт #1, английский
- `organic` — прямой заход без метки (дефолт)

**Ограничения:**
- Только латиница, цифры, `_` и `-`
- Максимум 40 символов
- Хитрые символы (`?`, `&`, пробелы) автоматически удаляются

### Веди таблицу

Заведи Google Sheet с двумя колонками:

| Аккаунт (как ты его помнишь) | URL для сторис |
|---|---|
| Instagram @anna_50plus | `https://velvet-soft.pages.dev/?s=ig-anna_de` |
| Facebook page «Quiet Mornings» | `https://velvet-soft.pages.dev/?s=fb-quiet_de` |

Перед публикацией каждой сторис копируешь URL из таблицы — не вспоминаешь по памяти.

## Распределение нагрузки на Meta

### Что Meta считает «нагрузкой»

Meta смотрит на **домен** (`velvet-soft.pages.dev`), не на URL целиком. Поэтому:

| Способ распределения | Эффективность |
|---|---|
| Разные `?s=...` метки | ⭐ слабо (помогает только для preview cache) |
| Разные пути (`/de/`, `/pl/`) | ⭐⭐ средне |
| Разные домены | ⭐⭐⭐⭐⭐ сильно (это РАЗНЫЕ сайты для Meta) |

**Реальное размывание нагрузки = ротация доменов.**

### Как делать ротацию доменов

1. Купи 2-3 дополнительных домена (порядка `meet-anna.com`, `quietmornings.com`, `lifeafter50.co`)
2. Используй один и тот же код (форкни Pages-проект или подключи разные домены к одному репозиторию)
3. Распределяй трафик: на одном аккаунте используй домен A, на другом — B, на третьем — C
4. Когда один домен «обжигается» (ссылки начинают резать) — переключаешь все аккаунты на запасной

**Минимум для боевого арбитража:** 2 домена в активной ротации + 1-2 в резерве.

### Tracking всё равно через `?s=`

Метки `?s=fb-page_de` остаются актуальными для **внутренней статистики партнёрки**. Они помогают тебе понимать какой аккаунт конвертит лучше. Просто не путай это с защитой от Meta.

## Cloaking — включение и выключение

### Что такое cloaking

Технология «два разных контента»: ботам Meta/Google показываем нейтральную страницу про «slow living journal», людям — настоящий age-gate.

### ⚠️ Важные предупреждения

- **Cloaking нарушает Terms of Service** Meta, Google и других платформ. Это серая зона.
- При обнаружении автоматическим классификатором: ссылка режется, аккаунт **может** быть забанен (15-25% при manual review).
- Cloaking **не панацея** — твой soft-mainstream gate является первой линией защиты.

### Когда включать

**НЕ включай если:**
- Ты только запустил трафик
- Ссылки живут больше 7 дней без cloaking
- У тебя нет 2-3 запасных аккаунтов

**Включай если:**
- Ссылки режут за 1-3 дня
- Есть backup-аккаунты готовые к замене основного
- Готов к риску потерять аккаунт

### Как включить

Открой `functions/_middleware.js`. Найди строку:

```javascript
const CLOAKING_ENABLED = false;
```

Поменяй на:

```javascript
const CLOAKING_ENABLED = true;
```

Сохрани, push (или re-deploy). Через 30-60 секунд cloaking активен.

### Как выключить

Та же строка → `false`. Push.

### Что меняется при включённом cloaking

- Бот с известным User-Agent → видит «Mindful Mornings» страницу
- Бот с известного Meta IP → видит «Mindful Mornings» страницу
- Обычный юзер → видит твой age-gate как обычно

## Логирование ботов и сбор базы

### Зачем

Чтобы понимать **какие именно боты тебя проверяют** и поддерживать актуальную базу для cloaking.

### Как работает

По умолчанию `BOT_LOGGING_ENABLED = true` в `_middleware.js`. Это значит:
- Каждый бот логируется (User-Agent, IP, страна, URL)
- 5% обычных юзеров тоже логируется (для baseline)
- Все логи идут в Cloudflare console (Real-time Logs)

### Как смотреть логи

1. Cloudflare Dashboard → Workers & Pages → `velvet-soft`
2. Tab **Logs** → **Real-time Logs**
3. Нажимаешь **Begin log stream**
4. В реальном времени видишь все запросы. Каждый бот появляется как JSON:

```json
{"type":"bot_detected","reason":"user_agent","ip":"66.220.144.10","ua":"facebookexternalhit/1.1","country":"US","url":"https://velvet-soft.pages.dev/?s=fb-page_de","time":"2026-06-15T15:30:00.000Z"}
```

### Как собрать базу ботов

**Метод 1 — пассивное наблюдение:**

Просто публикуешь сторис на реальных аккаунтах с разными `?s=...` метками. В первые 5-30 минут после публикации в логи прилетят User-Agent ботов которые тебя сканируют. Записывай их в `BOT_PATTERNS` в `_middleware.js`.

**Метод 2 — honeypot:**

1. Создай ссылку которую **никуда не публикуешь**: `https://velvet-soft.pages.dev/?s=honeypot-test-xyz`
2. Опубликуй её в **тестовой сторис** на тестовом аккаунте IG/FB
3. Жди 30-60 минут
4. Открываешь Real-time Logs → видишь всех ботов которые проверили ссылку
5. Добавляешь незнакомые User-Agent в `BOT_PATTERNS`

**Что добавлять в базу:**

Из логов:
- ✅ User-Agent содержит `bot`, `crawler`, `spider`, `headless` → точно бот
- ✅ User-Agent содержит название известного сервиса (`Meta-`, `FB-`, `Instagram-`) → бот
- ✅ IP в диапазоне Meta → бот (хотя у нас уже есть IP-фильтр)
- ⚠️ User-Agent имитирует обычный Chrome, но запросы быстрые/частые → возможно скрытый бот

### Обновление базы

Раз в 1-2 месяца:

1. Смотришь Real-time Logs за последние 2-3 дня
2. Фильтруешь по `"type":"bot_detected"` → собираешь уникальные User-Agent
3. Сравниваешь с `BOT_PATTERNS` в коде — добавляешь новые
4. Push, deploy

**Бонус: открытые базы на GitHub.**

Поищи репозиторий `monperrus/crawler-user-agents` — это поддерживаемый сообществом список ~1000 ботов в JSON. Раз в 1-2 месяца качаешь свежую версию, обогащаешь свой `BOT_PATTERNS`.

### Как отключить логирование

В `_middleware.js`:

```javascript
const BOT_LOGGING_ENABLED = false;
```

Когда логи не нужны (например, после того как база собрана) — отключи чтобы не засорять Cloudflare console.

## Подключение своего домена

⚠️ **Критически важно для боевого арбитража.** `*.pages.dev` уже слегка подозрителен для Meta.

### Покупка

Рекомендую **Porkbun** (~$10/год) или Namecheap.

**Не бери:**
- ❌ `.xyz`, `.bond`, `.online` — плохая репутация
- ❌ `.tk`, `.ml`, `.ga` — бесплатные, токсичны

**Бери:**
- ✅ `.com` — золотой стандарт
- ✅ `.co`, `.org` — приемлемо
- ✅ `.de`, `.pl` — если льёшь только в эти страны

**В названии домена НЕ используй:**
- ❌ `meet`, `dating`, `love`, `sexy`, `mature` — триггеры Meta
- ✅ нейтральные слова — `quietmornings`, `silentcoffee`, `slowsunday`, `aprilroom`, `windowlight`

### Подключение к Cloudflare Pages

1. Cloudflare Pages → твой проект → **Custom domains**
2. **Set up a custom domain** → введи домен
3. Cloudflare покажет нужные DNS-записи. Лучший вариант — перенести DNS-сервера домена на Cloudflare (полное управление через дашборд). Альтернатива — добавить CNAME у регистратора.
4. SSL-сертификат выпустится за 5-30 минут автоматически
5. Готово, домен работает

### Никогда не делай

- ❌ Не публикуй `*.pages.dev` URL в сторис на боевых аккаунтах
- ❌ Не используй один домен на 5+ аккаунтах одновременно
- ❌ Не покупай домен с одним и тем же contact data на 10 разных доменов — Meta это связывает

## Проверка работы 302 и cloaking

### Проверка 302 редиректа

1. Открой `https://velvet-soft.pages.dev/?s=test123` в Chrome
2. Нажми F12 → tab **Network**
3. Нажми кнопку «I'm 18+ — show me →»
4. В Network увидишь:
   - `go?s=test123` → статус **302**
   - `aff_f?h=rA47F6&aff_sub5=test123` → запрос на партнёрку

Если 302 нет — функция не отработала, проверь что `functions/go.js` залит.

### Проверка cloaking (после включения)

**Шаг 1: Проверь что обычный юзер видит свой gate**

Открой `https://velvet-soft.pages.dev/` в обычном Chrome → должен видеть age-gate.

**Шаг 2: Симулируй бота через User-Agent**

В Chrome:
1. F12 → tab **Network** → иконка **⋮** (три точки) → **More tools** → **Network conditions**
2. Раздел **User agent** → uncheck «Use browser default»
3. Введи в поле: `facebookexternalhit/1.1 (+http://www.facebook.com/externalhit_uatext.php)`
4. Обнови страницу (Ctrl+F5)
5. Должен увидеть страницу «Mindful Mornings — Slow Living Journal»

Если видишь age-gate — cloaking НЕ работает. Проверь:
- `CLOAKING_ENABLED = true` в `_middleware.js`
- Файл `_middleware.js` в папке `functions/`
- Деплой прошёл успешно (Cloudflare Dashboard → Deployments → последний должен быть зелёным)

**Шаг 3: Проверь логирование**

1. Cloudflare Dashboard → твой проект → Logs → Real-time Logs → Begin log stream
2. Открой `https://velvet-soft.pages.dev/` ещё раз с UA `facebookexternalhit`
3. В логах должна появиться запись `"type":"bot_detected"`

## Безопасность аккаунтов

### До запуска

- [ ] Аккаунту IG/FB **минимум 6 месяцев**, лучше год+
- [ ] У аккаунта есть реальная история (посты, друзья, не похож на свежеслитый)
- [ ] Привязан реальный/чистый номер телефона (не из массового SMS-сервиса)
- [ ] Привязан чистый email (не Gmail с тысячью аккаунтов на одном IP)

### Backup стратегия

- [ ] **3 запасных аккаунта** готовы к замене основного
- [ ] У каждого аккаунта свой devices (телефон/эмулятор)
- [ ] У каждого аккаунта чистый IP при логине
- [ ] **2-3 запасных домена** деплойнуты и готовы (но не используются)

### Диверсификация

- [ ] Минимум 5 разных меток `?s=...` на один аккаунт (распределяй кампании)
- [ ] Не лей весь трафик в один день — растягивай по времени
- [ ] Чередуй сторис с/без ссылки (10 сторис в день, ссылки в 3-4)

### Сигналы что аккаунт под угрозой

- Охват сторис резко упал (-50% за 2 дня)
- Не можешь публиковать сторис со ссылкой («Не удалось опубликовать»)
- Появились warnings в Account Status
- IG/FB просит подтвердить телефон/email повторно

**При любом из этих сигналов** — приостанавливай этот аккаунт на 7-14 дней, переходи на backup.

### Технический

- [ ] Партнёрская ссылка в `functions/go.js` корректная

- [ ] 302 проверен в DevTools
- [ ] Cookie banner появляется через 4 секунды и закрывается
- [ ] Footer ссылки ведут на реальные страницы (Privacy/Terms/Cookies/Impressum) — **не на `#`**

### Контентный

- [ ] Тексты на DE проверены носителем (или хотя бы DeepL + грамматика)
- [ ] Тексты на PL проверены носителем
- [ ] H1 заголовок соответствует тону Reels креативов
- [ ] CTA call-to-action не выглядит как age-confirmation, а как приглашение

### Защитный

- [ ] Custom domain подключён (не `*.pages.dev`)
- [ ] `noindex,nofollow` в meta robots
- [ ] Backup домены готовы (минимум 2)
- [ ] Backup аккаунты готовы (минимум 3)
- [ ] Cloaking выключен на старте (`CLOAKING_ENABLED = false`)
- [ ] Bot logging включён (`BOT_LOGGING_ENABLED = true`)
- [ ] Таблица меток `?s=` ведётся

### Юридический (для DE/PL/AT)

- [ ] Impressum заполнен (для DE/AT — обязательно)
- [ ] Privacy policy готов
- [ ] Cookie banner работает
- [ ] Возраст 18+ подтверждается явно
